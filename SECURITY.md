# Security Policy

## Supported version

The v0.1.x Preview line receives security review while it is the latest published line. Older previews may be retired instead of patched when their data or release contracts are incompatible.

## Report privately

Do not open a public issue for a suspected vulnerability. Report it privately through the repository security-reporting feature once that channel is enabled. Until then, contact the maintainer through an already established private channel and provide only the minimum reproducible details.

Never submit real enterprise reports, confidential process data, personal information, credentials, access tokens, private keys, or unredacted logs. Replace all business data with fictional values before sharing a reproducer.

## What to include

Describe the affected version, operating system, Python version, entry point, expected behavior, observed behavior, and a minimal fictional reproducer. Explain whether the issue concerns file export, archive inspection, source scanning, model validation, command execution, or documentation.

## Response process

A maintainer will acknowledge receipt, assess impact and reproducibility, coordinate a correction, add a regression test, and determine whether a new release is required. Publication timing will be coordinated so users can upgrade before exploit details are broadly disclosed.

Security reports do not authorize access to systems or data that you do not own or have permission to test.
